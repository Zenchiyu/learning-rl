What is the difference between epsilon-greedy and epsilon-soft policies?

February 27, 2020

Learn more about Reinforcement Learning in my course (75% off more more when you use the following link):

Artificial Intelligence: Reinforcement Learning in Python

A common question I get in my Reinforcement Learning class is:

“What is the difference between epsilon-greedy and epsilon-soft policies?”

At first glance, it may seem that these are the same thing. At times, in Sutton & Barto, it seems these 2 terms are used interchangeably.

Here’s the difference.

An epsilon-soft (ε−softε−soft) policy is any policy where the probability of all actions given a state ss is greater than some minimum value, specifically:

π(a|s)≥ε|A(s)|,∀a∈A(s)π(a|s)≥ε|A(s)|,∀a∈A(s)

The epsilon-greedy (ε−greedyε−greedy) policy is a specific instance of an epsilon-soft policy.

Specifically, the epsilon-greedy policy can be defined as epsilon-greedy with respect to the action-value Q(s,a)Q(s,a).

Let a∗a∗ be the greedy action with respect to Q(s,a)Q(s,a), so that:

a∗=argmaxaQ(s,a)a∗=arg⁡maxaQ(s,a)

Then the epsilon-greedy policy assigns the following probabilities to the actions in A(s)A(s):

π(a|s)π(a|s)==1–ε+ε|A(s)|,ε|A(s)|,a=a≠a∗a∗π(a|s)=1–ε+ε|A(s)|,a=a∗π(a|s)=ε|A(s)|,a≠a∗

This can be accomplished using the following code:

def epsilon_greedy(Q, s, epsilon):
  if np.random.random() < epsilon:
    return np.random.choice(action_space)
  else:
    return np.argmax(Q[s, :])

This assumes that Q is a Numpy-like array with 2 dimensions corresponding to the possible states and actions.

Based on this, we can see that all epsilon-greedy policies are epsilon-soft policies, but not all epsilon-soft policies are epsilon-greedy policies.
